## ステージ1: BuyMore - タイマーと隠し鍵

### テーマ
オンラインショッピングサイトからの退会

### ストーリー上の位置づけ
「退会への鍵を見つける」- 退会の入り口を探す

### ゲーム内容

#### 画面構成
- ヘッダー: サイトタイトル「BuyMore」
- 右上: 1分カウントダウンタイマー（常時減少）
- メインエリア: 大量の商品カード（20個程度）をごちゃごちゃと配置
- フッター: 会社概要、利用規約などのリンク

#### ギミック
1. タイマーをクリックすると、タイマーの下に鍵アイコンが出現
2. 鍵アイコンをクリックでステージクリア

#### 風刺ポイント
- プレイヤーは「時間制限」と思い込み焦る
- 実は時間制限ではなく、タイマー自体がヒント
- 「時間が減っていく不安」を利用したダークパターンの逆利用

#### 実装詳細
- タイマー: useEffectで1秒ごとに減少（00:00になっても何も起きない）
- 鍵の出現: タイマークリックでstate変更、アニメーションで鍵が下にスライド
- 商品カード: 既存の4つを20個程度に増やし、画面をごちゃごちゃさせる

## ステージ2: Info-Sphere - 吹き出しの中の鍵穴

### テーマ
ニュースアプリからの退会

### ストーリー上の位置づけ
「鍵穴を見つける」- ステージ1で手に入れた鍵を使う場所を探す

### ゲーム内容

#### 画面構成
- ヘッダー: サイトタイトル「Info-Sphere」
- メインエリア: 大量のフェイク鍵穴アイコン（🔓など）を配置
- 動く吹き出し: 複数の吹き出しが画面を動き回る

#### ギミック
1. 画面上に15個程度のフェイク鍵穴を配置（クリックしても無反応）
2. 動く吹き出しが3〜5個、ランダムに移動
3. 吹き出しのテキストの中に「。」（句点）があり、これが本物の鍵穴
4. 吹き出し内の「。」をクリックでステージクリア

#### 風刺ポイント
- 「ヒントがあるはず」と吹き出しを読むが、実はヒント自体が答え
- 句点という些細な記号が本物という意外性
- 動く吹き出しで視界を妨害する悪質UI

#### 実装詳細
- 吹き出し: CSS animationまたはJSでランダムに動く
- フェイク鍵穴: クリックでちょっとしたエフェクト（間違い演出）
- 本物の鍵穴: 吹き出し内の`<span>`タグで「。」をクリッカブルに

## ステージ5: Everything - 規約スクロール地獄

### テーマ
総合プラットフォームからの退会

### ストーリー上の位置づけ
最終ステージ - 「ついに退会できる...？」という期待を裏切る

### ゲーム内容

#### 画面構成
- タイトル: 「退会に関する重要事項」
- サブタイトル: 「最後までお読みいただき、同意の上で退会手続きを進めてください」
- メインエリア: スクロール可能な長文テキストボックス（高さ制限あり）
- ボトム: 「同意して退会」ボタン（スクロール最下部に到達で出現）

#### ギミック
1. 長文の利用規約風テキストを表示
2. プレイヤーは「最後まで読まないと...」と思い込む
3. 実は規約文の途中（例: 第7条）に「退会」という文字があり、これがクリッカブル
4. 「退会」をクリックでステージクリア
5. 最下部の「同意して退会」ボタンはフェイク（クリックしても何も起きない、または「処理中...」で止まる）

#### 風刺ポイント
- 「最後まで読め」という指示に従う真面目な人ほど答えに気づかない
- 規約を読まずに途中をクリックする人の方が早くクリア
- 長文規約の中に重要な情報を隠す悪質な手法を皮肉る
- 「同意」を求めるUIが実は罠

#### 実装詳細
- 長文テキスト: 実際の利用規約風の文章を生成（1000文字程度）
- スクロール検知: onScrollイベントで最下部到達を判定
- クリッカブル「退会」: 規約文中の特定の「退会」という文字を`<span>`で囲み、onClickイベント
- フェイクボタン: クリックで「処理中...」と表示し、数秒後に「エラーが発生しました」など
