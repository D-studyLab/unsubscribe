# 要件定義書：『The Unsubscribe』

## 1. プロジェクト概要

### 1.1 プロジェクト名
『The Unsubscribe』（ジ・アンサブスクライブ）

### 1.2 コンセプト
退会することに特化した体験型風刺ゲーム。プレイヤーは様々な架空のWebサービスから退会を試みるが、巧妙なダークパターンによって引き止められる。全5ステージを突破し、すべてのサービスから退会することが目的。

### 1.3 ターゲット
- パズルや脱出ゲームが好きな人
- 社会風刺やブラックユーモアに関心がある人
- Webサービスのダークパターンに不満を感じたことがある人

## 2. 技術要件

### 2.1 技術スタック
- **フロントエンド**: React (Create React App)
- **言語**: TypeScript（型安全性を重視）
- **デプロイ**: Firebase Hosting
- **レンダリング方式**: CSR (Client Side Rendering)

### 2.2 Firebaseの使用範囲
- Firebase Hosting（デプロイとホスティング）のみ
- Firestore、Firebase Authentication等は**使用しない**
- 完全にクライアントサイドで完結

### 2.3 データ永続化
- データベースは使用しない
- プレイヤーの進捗やスコアは保存しない
- 必要に応じてローカルストレージを使用（例：ニックネーム、設定等）

## 3. 機能要件

### 3.1 コア機能

#### 3.1.1 ゲームフロー
1. **スタート画面**
   - タイトル表示
   - ニックネーム入力欄（デフォルトでランダムな名前が入力済み）
   - 「ゲームスタート」ボタン

2. **ステージ進行**
   - 全5ステージを順番通りにプレイ
   - ステージ選択機能は**なし**
   - 各ステージクリア後、次のステージへ自動遷移

3. **結果画面**
   - クリア時間の表示
   - ニックネームの表示
   - SNS共有ボタン

#### 3.1.2 ニックネーム機能
- ゲーム開始時、入力欄にランダムな名前を自動生成して表示
- プレイヤーは変更可能（任意）
- 面倒な人はそのまま「決定」ボタンで進める
- 結果画面でニックネームとスコアをセットで表示

#### 3.1.3 SNS共有機能
- 結果画面に共有ボタンを配置
- 他のゲームでよくあるUI/UXを参考に実装
- Twitter(X)への共有を優先
- 共有内容：プレイ結果（クリア時間、ニックネーム等）

### 3.2 ステージ詳細

#### ステージ1：オンラインショッピングサイト『BuyMore』
- **ダークパターン**:
  - 情報隠蔽（退会リンクがフッターの小さな文字）
  - 偽の階層構造（目立たないデザイン）

#### ステージ2：ニュースアプリ『Info-Sphere』
- **ダークパターン**:
  - 恥の植え付け（不安や劣等感を煽るメッセージ）
  - ひっかけ質問（「退会しない」が強調表示）

#### ステージ3：クリエイター支援SNS『Fan-Circle』
- **ダークパターン**:
  - 執拗なアンケート（長文の必須回答）
  - 社会的証明（人間関係に訴えかける引き止め）

#### ステージ4：格安航空券予約サイト『Sky-Cheap』
- **ダークパターン**:
  - こっそりオプション（有料サポートが勝手にチェック済み）
  - 偽のUI・無限ループ（ボタンを押すとトップページに戻る）

#### ステージ5：総合エンタメプラットフォーム『Everything』
- **ダークパターン**:
  - 不滅のアカウント（退会ページが存在しない）
  - オフライン誘導（電話必須、受付時間が異常に短い）
  - 謎解き要素（特定のタイミングで特定の番号を押す）

### 3.3 優先度の低い機能（将来実装）
- ヒント機能
- 英語版対応
- ランキング機能（**現時点では実装しない**）

## 4. 非機能要件

### 4.1 対応デバイス
- **優先**: PC（デスクトップ・ノートPC）
- **将来対応**: モバイル（スマートフォン・タブレット）
- レスポンシブデザインを考慮した設計を行うが、まずはPC版を完成させる

### 4.2 対応ブラウザ
- Chrome（最新版）
- Firefox（最新版）
- Safari（最新版）
- Edge（最新版）

### 4.3 パフォーマンス
- ページ読み込み時間：3秒以内
- ステージ遷移：スムーズなアニメーション
- 軽量な実装を心がける

### 4.4 アクセシビリティ
- キーボード操作対応
- スクリーンリーダー対応（最低限）
- 色覚異常への配慮

## 5. UI/UXガイドライン

### 5.1 デザインコンセプト
- 各ステージは実在のWebサービスをリアルに模倣
- ダークパターンを体験させるため、意図的に「使いにくい」UI
- 風刺的でブラックユーモアのあるメッセージ

### 5.2 トーンアンドマナー
- 皮肉たっぷりの文章
- プレイヤーに「悔しい」「理不尽だ」と感じさせる
- 社会問題への気づきを促すメッセージング
